Hexo + GitHub Pages 个人博客搭建

更新于: 12 Feb, 2026
|

这篇文章记录第一次从零搭建 Hexo + GitHub Pages 个人博客的过程。重点不是复述所有官方步骤，而是把容易踩坑的地方记录下来。

环境准备

工具用途
Node.js提供 Hexo 运行环境
Git管理源码和部署
Hexo CLI创建、生成、预览博客
GitHub Pages托管静态站点

检查 Node.js 和 npm：

node -v
npm -v

安装 Hexo CLI：

npm install -g hexo-cli

创建博客项目

hexo init blog
cd blog
npm install

本地启动预览：

hexo clean
hexo generate
hexo server

浏览器打开 http://localhost:4000，能看到页面说明本地生成正常。

GitHub Pages 仓库

用户站仓库命名格式是：

<username>.github.io

例如：

Shiaoming123.github.io

这个仓库的 main 分支通常直接作为静态站发布源。也就是说，最终进入这个仓库的应该是 HTML、CSS、JS、图片等构建产物，而不是随手修改的临时页面。

配置部署

安装部署插件：

npm install hexo-deployer-git --save

在 _config.yml 中配置：

deploy:
  type: git
  repo: git@github.com:Shiaoming123/Shiaoming123.github.io.git
  branch: main

发布命令：

hexo clean
hexo generate
hexo deploy

主题切换

早期可以尝试 yilia、NexT、Butterfly 等主题。切换主题前要先确认主题和 Hexo 版本兼容。

以 yilia 为例：

git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

然后修改 _config.yml：

theme: yilia

切换主题后重新生成：

hexo clean
hexo generate
hexo server

图片管理

博客图片可以走本地目录，也可以走图床。为了迁移和备份更稳定，建议优先保证本地源码里有一份可用图片。

如果使用 GitHub 图床，PicGo 配置里通常需要：

配置项说明
仓库名例如 Shiaoming123/Picgo_image
分支名通常是 main
存储路径例如 img/
自定义域名可使用 raw 链接或 CDN

不要把 token 写进文章或仓库。文章里只记录配置项含义，不保存真实密钥。

常见问题

部署后页面没更新

先检查 GitHub Pages 仓库是否有新 commit，再等待 Pages 刷新。如果本地生成正常但线上没变，通常是 deploy 没有推送成功。

图片在线上不显示

检查三件事：

  1. 图片文件是否已提交或上传
  2. Markdown 里的路径是否和部署后的路径一致
  3. 图片链接是否需要 URL 编码

主题配置不生效

先执行清理再生成：

hexo clean
hexo generate

如果仍不生效，再检查主题配置文件和 Hexo 主配置是否写在正确位置。
