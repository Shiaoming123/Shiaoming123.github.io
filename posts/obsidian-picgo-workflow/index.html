前言

作为技术博主，图片管理一直是个头疼的问题：

⚠️ 传统方式的问题 - 写作时需要手动上传图片到图床 - 逐个替换文章中的图片链接（容易遗漏） - 本地预览速度慢（依赖图床网络） - 重复劳动，浪费时间

经过实践，我找到了一个完美的解决方案：Obsidian + PicGo 自动化工作流。

🐛 常见问题

Q1: 图片上传失败？

A: 检查以下几点：

  1. PicGo 图床配置是否正确
  2. GitHub Token 是否有效
  3. 网络连接是否稳定
  4. 图片文件是否存在

Q2: 链接没有被替换？

A: 可能原因：

  • 图片上传失败（脚本跳过失败的图片）
  • 文件名不匹配

Q3: 部署后图片不显示？

A: 排查步骤：

  1. 检查文章中的 URL 是否正确
  2. 访问图床 URL 验证图片可访问
  3. 清理浏览器缓存
