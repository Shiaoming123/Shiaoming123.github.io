🔥 今日头条（5 个最重磅的）

1. 韩国启动史上最大规模产业投资计划：5760亿美元投向半导体与AI

  • 一句话：韩国政府宣布迄今规模最大的半导体与AI产业投资计划，总投资超5760亿美元，三星和SK海力士将在西南地区新建两个芯片制造基地，目标五年内将DRAM产能翻倍。
  • 为什么重要：这是单一国家在半导体领域的最大规模投资之一，直接回应全球AI算力短缺和存储芯片价格暴涨的现状。三星、SK海力士负责人与总统李在明一同出席发布会，战略意义不言而喻。
  • 来源：Ars Technica / 36氪

2. Rocketlab 收购 Iridium，打造全栈卫星通信公司

  • 一句话：Rocketlab 宣布收购卫星通信巨头 Iridium，交易金额未披露，将整合火箭发射与卫星通信网络，成为”全栈”航天通信公司。
  • 为什么重要：Rocketlab 从火箭发射商转型为卫星运营商+发射服务商，标志着航天行业垂直整合的新趋势。HN 上 355 点赞，221 条评论，社区热议。（来源：Rocketlab / HN）

3. 涉嫌操纵内存价格，三星、SK海力士、美光遭集体诉讼

  • 一句话：美国消费者和小型企业于6月25日在加州联邦法院对三星、SK海力士和美光提起集体诉讼，指控三家巨头合谋限制DRAM供应、操纵价格，加剧全球内存短缺危机。
  • 为什么重要：全球DRAM市场几乎被这三家垄断。苹果和多家手机厂商已因存储芯片成本上涨提价。若诉讼成立，可能重塑全球存储芯片定价格局。（来源：36氪/财联社）

4. 美国最高法院裁定：geofence 搜查令需受宪法保护

  • 一句话：美国最高法院裁定，执法机构获取手机地理围栏数据（geofence warrant）必须受宪法第四修正案保护，实质性地限制了大规模位置追踪的权力。
  • 为什么重要：这一定义将为数字时代的隐私保护树立新判例。HN 上获得 419 点赞、195 条评论，是当天最受关注的法律/科技交叉新闻。
  • 来源：The Guardian / HN

5. DeepSeek V4 正式版官宣7月上线，引入峰谷定价机制

  • 一句话：DeepSeek 团队宣布 V4 正式版计划 7 月中旬上线，同时将调整 API 定价策略，高峰时段（9:00-12:00、14:00-18:00）价格为平时的 2 倍。
  • 为什么重要：这是国产大模型首次引入”峰谷电价”式动态定价，反映算力供需矛盾的加剧。同时 V4 版本性能提升，是否会拉高 API 使用门槛值得关注。
  • 来源：36氪

🤖 AI 进展（5 个）

1. Qwen 3.6 27B：本地开发的「甜蜜点」模型

  • 一句话：开发者实测报告称，通义千问 Qwen 3.6 27B 模型在本地开发场景下表现出色，性能与精度达到理想平衡。
  • 为什么重要：580 点赞、498 条评论——HN 当天最高热度故事。说明开发者社区对”能跑在本地的高质量模型”有巨大需求，27B 成为兼顾性能与资源消耗的黄金参数规模。
  • 来源：HN

2. 马斯克宣布 Grok 4.5 内部私测：性能或超越 Claude Opus

  • 一句话：马斯克发文称 Grok 4.5 已在 SpaceX 和特斯拉内部开启 Beta 测试，基于 1.5 万亿参数 V9 基础模型，加入 Cursor 数据训练，早期评测接近甚至超越 Claude Opus。
  • 为什么重要：xAI 加速追赶 Anthropic 和 OpenAI。马斯克还表示 SpaceX 今年每月都将发布完全从头训练的新模型，这可能意味着 xAI 的算力储备大幅提升。
  • 来源：36氪/界面新闻

3. vLLM 发布 Micro-Agent：在模型 API 内部协作击败前沿模型

  • 一句话：vLLM 团队提出 Micro-Agent 概念，通过多个小型模型在 API 内部协作，在多项基准测试中击败了 GPT-4o、Claude Opus 等前沿大模型。
  • 为什么重要：这是 Agent 领域的新方向——不是用更大的模型，而是用多个小模型协作实现超越。降低了 Agent 系统的成本门槛。
  • 来源：vLLM Blog / HN

4. 百度开源 Unlimited OCR 四榜登顶

  • 一句话：百度发布并开源端到端 OCR 模型 Unlimited OCR（3B 参数，推理仅激活 570M），在 GitHub 和 HuggingFace 四大榜单登顶，OmniDocBench 测试以 93.92% 刷新纪录。
  • 为什么重要：OCR 这个看似”古老”的方向被大模型技术重新激活。570M 激活参数即可达到 SOTA，说明端侧部署 OCR 成为可能。
  • 来源：36氪

5. Ornith-1.0：自我改进的开源 Agent 编程模型

  • 一句话：DeepReinforce AI 发布 Ornith-1.0，一个能自我改进（self-scaffolding）的开源 Agent 编程模型，可用于自动化代码生成和调试。
  • 为什么重要：144 点赞 HN 热门。开源 Agent 编程模型逐步成熟，让开发者可以本地运行并自定义编码 Agent，与 Claude Code / Cursor 等商业产品形成竞争。
  • 来源：GitHub / HN

💻 开发者/工具（4 个）

1. Open Memory Protocol：为 Claude、ChatGPT、Cursor 提供统一记忆存储

  • 一句话：开源项目 Open Memory Protocol 旨在为多个 AI 工具（Claude、ChatGPT、Cursor 等）提供统一的外部记忆存储层，让不同 AI 共享上下文记忆。
  • 为什么重要：多工具之间的”记忆孤岛”是当前 AI 工作流的核心痛点。一个开源协议有望打破这一壁垒。
  • 来源：GitHub / HN

2. Apple Neural Engine 架构、编程与性能深度论文

  • 一句话：一篇关于 Apple Neural Engine（ANE）的深度技术论文在 arXiv 发布，详细解析了 ANE 的硬件架构、编程模型和性能特征。
  • 为什么重要：苹果在端侧 AI 芯片方面一直相对封闭。这篇 108 点赞的 HN 热门论文为开发者理解并利用 ANE 提供了权威参考。
  • 来源：arXiv / HN

3. 什么是 CUDA Kernel 运行时的完整过程？

  • 一句话：一篇深度技术博文，从头到尾讲解 GPU 从收到 CUDA kernel 指令到执行完成的完整软硬件链路，包括驱动调度、SM 分配、warp 调度等。
  • 为什么重要：199 点赞 HN 热门。对想要理解 GPU 底层原理的 AI 工程师来说是不可多得的技术长文。
  • 来源：HN

4. JumpServer：开源特权访问管理平台

  • 一句话：JumpServer 是一款开源特权访问管理（PAM）工具，支持堡垒机、会话审计、权限控制等功能。
  • 为什么重要：53 点赞 HN 热门。开源 PAM 方案对中小企业和开发者团队极具吸引力，尤其是在安全合规要求日益严格的当下。
  • 来源：GitHub / HN
