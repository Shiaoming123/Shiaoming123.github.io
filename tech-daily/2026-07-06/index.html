🔥 今日头条（5 个最重磅的）

1. 美团 LongCat-2.0 完全开源（MIT 许可），1.6T MoE 模型开放权重与推理代码

  • 一句话：美团宣布 LongCat-2.0 以 MIT 许可证完全开源，公开 1.6T 总参数量（约 48B 活跃参数）的 MoE 模型权重与推理代码，支持 1M token 上下文。
  • 为什么重要：这是目前国内最大的全开源 MoE 模型之一。LongCat-2.0 在多项基准中超越 GPT-5.5（SWE-bench Pro 59.5 vs 58.6），原生集成 Claude Code、OpenClaw、Hermes Agent，支持 GPU 与 NPU 部署。MIT 许可无任何限制，开源社区可直接使用、商用和二次开发。
  • 来源：AIHOT / Meituan LongCat @ X

2. 腾讯发布 295B 参数开源 MoE 模型 Hy3（非预览版正式版）

  • 一句话：腾讯正式发布 Hy3，295B 参数 MoE 架构，21B 激活参数 + 3.8B MTP 层，盲测得分 2.67/4 超越 GLM-5.1 的 2.51/4。
  • 为什么重要：Hy3 的前端开发、CI/CD 及数据存储能力尤为突出，权重已上架 HuggingFace。继 4 月底 Hy3 Preview 之后，这是腾讯在大模型开源赛道的又一重注。腾讯元宝已同步接入。
  • 来源：AIHOT / TestingCatalog

3. 字节跳动、阿里巴巴等因新规强制关闭 AI 伴侣/人性化聊天功能

  • 一句话：因中国监管新规要求，字节跳动豆包（月活超 3 亿）7 月 15 日下线个性化功能，阿里通义千问 7 月 10 日撤下人性化智能体，腾讯元宝 6 月已执行。
  • 为什么重要：新规要求 AI 伴侣产品警告过度使用、检测成瘾行为，禁止引发未成年人极端情绪或培养依赖排挤现实关系，同时禁止在敏感对话数据上训练。这是中国在 AI 情感陪伴领域最强监管动作，直接影响数亿用户的交互体验，也预示着 AI 伴侣赛道的合规拐点。
  • 来源：AIHOT / The Decoder

4. NVIDIA Kyber NVL144 推迟至 2028 年，亚洲供应商股价下跌

  • 一句话：NVIDIA Kyber NVL144 机架延迟超 12 个月至 2028 年；备选设计 NVL72x2 被取消；更强 Rubin Ultra 芯片（四 die 版本）也被砍，仅保留双 die 版本，性能约减半。
  • 为什么重要：NVIDIA 下一代旗舰硬件的大幅延迟给 AMD MI500X 和 Google TPUv8i 等竞品创造了市场窗口。亚洲供应商股价短期承压，但分析师认为下跌主要由获利回吐驱动，并非 AI 整体支出放缓的信号。
  • 来源：AIHOT / The Decoder

5. AWS 关停 Mechanical Turk，经典”人工人工智能”谢幕

  • 一句话：Amazon Web Services 宣布关停 Mechanical Turk 平台，同时 SageMaker Ground Truth 和 Amazon Augmented AI 也同日关闭新客户注册。
  • 为什么重要：Mechanical Turk 于 2005 年推出，口号”Artificial Artificial Intelligence”，是众包数据标注和人工验证的鼻祖平台。2023 年研究发现众包工人开始使用语言模型完成任务，削弱了其”人类数据源”的价值。如今 AI 实验室转向 Scale AI、Surge AI 等专业供应商。这是一段互联网历史的终点。
  • 来源：AIHOT / The Decoder

🤖 AI 进展（5 个）

1. AI 颠覆初级程序员就业市场：斯坦福数据揭示 22-25 岁开发者就业锐减 19%

  • 一句话：斯坦福数字经济实验室发现，美国 22-25 岁软件开发人员就业较 2022 年峰值下降 19%，入门级岗位招聘减少 28%，计算机科学毕业生失业率达 6.1%（高于文科）。
  • 为什么重要：核心推手是 2024-2025 年兴起的 Agentic Programming（智能体编程）。GitHub 一年新增 3600 万账号，80% 新用户一周内使用 Copilot。编程工作总量增长 4.4%，但全部来自年长群体。“初级程序员”头衔正在消亡，对职涯规划和教育体系构成深刻冲击。
  • 来源：AIHOT / Hacker News（buzzing.cc 翻译）

2. 研究命名”Ghost Memory”：AI 智能体重复已过时事实的系统性问题

  • 一句话：研究人员定义并命名了”Ghost Memory”现象——长期运行的 AI 智能体记忆库中同时存在旧事实、当前事实和过渡期记录，导致答案模型出错。提出 A-TMA（状态感知覆盖层）解决方案。
  • 为什么重要：这是 Agent 系统落地中的关键痛点——当 Agent 持续运行数周/数月，它可能仍然自信地引用用户早已改变的信息。该研究为持久化 Agent 的记忆管理提供了系统性评估框架和修复方案。
  • 来源：AIHOT / @omarsar0

3. 前沿实验室加速模型发布节奏，转向滚动发布模式

  • 一句话：前沿 AI 实验室自 2025 年中以来已转向滚动发布模式，模型迭代频率显著加快，不再遵循传统的”大版本发布”节奏。
  • 为什么重要：模型发布节奏的变化意味着开发者和企业需要适应”持续更新”的生态，而非等待重大版本。这也加剧了开源社区追赶前沿的难度。
  • 来源：AIHOT / @kimmonismus

4. Google DeepMind COrigami：用 Gemini 与强化学习协同设计折纸

  • 一句话：Google DeepMind 展示 COrigami 项目，使用 Gemini 模型与强化学习协同，自动设计复杂的折纸结构。
  • 为什么重要：这是 AI 在物理结构设计领域的又一突破——将大语言模型的空间推理能力与 RL 的优化能力结合，可应用于柔性机器人、可展开结构、医疗器械等领域。
  • 来源：AIHOT / @fofrAI

5. IBM 宣布首次在量子计算机上实现聚变材料计算

  • 一句话：IBM 宣布在量子计算机上首次成功完成聚变反应堆材料的计算模拟，标志着量子计算在能源领域的里程碑式应用。
  • 为什么重要：聚变能被视为终极清洁能源，而材料计算是其核心瓶颈之一。IBM 的成果证明量子计算已能在实用层面推进聚变研究，虽然距离商用仍远，但方向意义重大。
  • 来源：AIHOT

💻 开发者/工具（5 个）

1. GitHub Trending 今日爆火：asgeirtj/system_prompts_leaks（★50,886）

  • 一句话：大规模 AI 系统提示词泄露仓库，包含 Claude Fable 5、Opus 4.8、Claude Code、Claude Design、ChatGPT 5.5 Thinking、GPT 5.5 Instant、Codex、Gemini 3.5 Flash 等模型的系统提示词。
  • 为什么重要：这是目前最全的 AI 模型 system prompt 泄露集合，对了解各大模型厂商的提示工程策略、安全设计和行为约束机制具有极高的研究价值。
  • 来源：GitHub Trending

2. addyosmani/agent-skills（★70,389）— AI Agent 生产级技能集

  • 一句话：由 Google Chrome 团队 Addy Osmani 维护的生产级 AI Agent Skills 集合，提供工程级的技能定义供 AI 编码 Agent 使用。
  • 为什么重要：从开发者工具到专业技能，这个仓库正在成为 Agentic Coding 生态的”技能市场”标准库之一，对 Claude Code、Codex、Cursor 等工具的使用者极具参考价值。
  • 来源：GitHub Trending

3. 阿里开源向量数据库 Zvec（alibaba/zvec）登顶 GitHub Trending

  • 一句话：阿里巴巴开源轻量极速进程内向量数据库 Zvec（C++ 实现），今日获得 ★13,216。
  • 为什么重要：Zvec 定位”轻量、闪电般快、进程内”，适合在 AI 应用中进行本地向量检索。阿里在基础软件开源领域的持续投入值得关注。
  • 来源：GitHub Trending

4. openai/codex-plugin-cc（★26,069）— 让 Claude Code 调用 Codex

  • 一句话：OpenAI 官方发布的 Codex Plugin for Claude Code，允许用户在 Claude Code 中调用 Codex 来审查代码或委派任务。
  • 为什么重要：这是两大 AI 编码工具之间前所未有的互操作性——Claude Code 用户现在可以直接利用 OpenAI Codex 的能力，标志着 AI 编码工具生态开始走向开放协作。
  • 来源：GitHub Trending

5. 免费开源 AI API 中转站监控系统 TokHub 上线

  • 一句话：开发者姚老师与 @vista8 开源了 TokHub——AI API 中转站监控及网关管理系统，配套监测网站 tokhub.me 同步上线。
  • 为什么重要：API 中转站在国内 AI 生态中扮演重要角色（整合国内外模型 API 资源），TokHub 的开源降低了自建中转站监控的门槛，对中小开发者和团队是实用工具。
  • 来源：AIHOT
