🔥 今日头条（5 个最重磅的）

1. Kimi K3 发布：2.8T 参数开源模型登顶前端编码榜，百万 token 上下文窗口

  • 一句话：月之暗面发布 Kimi K3，2.8 万亿参数 MoE 开源模型，在 Frontend Code Arena 以 1679 分登顶力压 Claude Fable 5 与 GPT-5.6 Sol，7 个前端赛道拿下 6 个第一，7 月 27 日开放权重。
  • 为什么重要：这是首个开源 3T 级模型，采用 Kimi Delta Attention 和 Attention Residuals 架构创新，API 定价对标前沿闭源模型（输入 15 美元/百万 tokens），放弃低价路线转向长上下文智能体编码赛道。定价逻辑的转变可能重塑竞争格局。
  • 来源：AIHOT / Moonshot AI: Kimi Blog

2. 苹果起诉 OpenAI：约 40 名前员工收到律师函，AI 硬件赛道首起重大商业秘密诉讼

  • 一句话：Apple 对 OpenAI 提起诉讼，指控其通过系统性挖角获取商业机密以加速 AI 硬件研发，已向约 40 名前员工发出律师函要求保存文件，称超 400 名前员工在 OpenAI 工作，寻求法院禁令阻止 OpenAI 使用苹果信息。
  • 为什么重要：正值 Apple 发布新版 Siri AI 公测版之际，外界猜测 Apple 意在用法律手段拖慢最强竞争对手。这场诉讼的走向可能比任何模型发布都更能定义下一阶段的 AI 竞争规则。
  • 来源：AIHOT / The Verge / IT之家

3. 欧盟裁定 Google 必须向竞争对手开放 Android 和 Search，直接影响 Gemini 等 AI 服务

  • 一句话：欧盟依据 DMA 裁定 Google 必须向竞争对手开放 Android 和 Google Search 关键部分，包括允许第三方 AI 助手和搜索引擎获得更大访问权限，可能削弱 Google 对两大核心平台的控制。
  • 为什么重要：对 Gemini 等 AI 助手的地位是实质冲击，做 AI 应用和搜索的创业者值得盯着后续执行细节。
  • 来源：AIHOT / The Verge / IT之家 / Ars Technica

4. 世界人工智能合作组织成立，29 国签署协定，总部设在上海

  • 一句话：7 月 16 日，世界人工智能合作组织协定签署仪式在上海举行，王毅代表中国签署，哈萨克斯坦、老挝、巴基斯坦等 29 国成为创始成员国，总部设在中国上海。
  • 为什么重要：全球 AI 治理从倡议走向落地，对中国 AI 企业的出海合规和参与国际规则制定是长期利好。
  • 来源：AIHOT / IT之家

5. 台积电上调 2026 年资本支出至 600~640 亿美元，A14 (1.4nm) 制程进展顺利

  • 一句话：台积电在 2026Q2 财报说明会将资本支出预测上调至 600~640 亿美元（此前约 560 亿），预计 2028 年量产的 A14 制程进展顺利，2026 全年美元收入增幅将超过 40%。
  • 为什么重要：直接宣告了 AI 算力在未来三年的供给曲线，做 AI 硬件的同行和投资人必须正视这轮扩产。
  • 来源：AIHOT / IT之家

🤖 AI 进展（8 个）

1. Claude Fable 5 在 CursorBench 达 72.9% 新高

Cursor 评估负责人确认，Claude Fable 5 在 CursorBench 以 Max effort 模式达到 72.9% 新高，在航天模拟器中仅凭一句提示自主规划并成功登月。（来源：AIHOT / Claude Blog）

2. Kimi K2.5 技术路线公开：MuonClip 优化器替代 Adam

月之暗面 CEO 杨植麟在 GTC 2026 提出用 MuonClip 优化器替代 Adam，数据利用效率提升近一倍；Kimi Linear 线性注意力在百万 Token 上下文下超越全注意力；Agent Swarm 已支持 300 个 Agent 并行工作。全部开源。（来源：AIHOT / 宝玉）

3. NVIDIA 发布 Nemotron 3 Embed 系列，8B 版本 RTEB 排名第一

8B-BF16 在 RTEB 基准以 78.46 平均 NDCG@10 排名第一，1B-NVFP4 在 Blackwell 上吞吐量比 BF16 高 2 倍，精度保留 99.5%。（来源：AIHOT / MarkTechPost）

4. 通义实验室 Wan-Streamer v0.2：端到端延迟仅 550ms

将”听、看、说、演”统一进单个 Transformer，输出分辨率提升至 640×368 @ 25FPS，采用 Thinker-Performer 双通路架构。（来源：AIHOT / 公众号：通义实验室）

5. Grok 推出 Automations 功能

xAI 为 Grok 引入定时/邮件触发自动执行任务功能，支持一次/每日/工作日/每周/每月/每年计划，邮件触发需 SuperGrok 订阅。（来源：AIHOT / xAI News）

6. 月之暗面发布 PerceptionBench 多模态视觉感知诊断基准

从 40+ 基准中模型失败案例归纳，包含 10 项原子感知能力和 3000 道验证题，所有测试模型准确率均未超过 60%，大量正确答案在重复提问时无法复现。（来源：AIHOT / Moonshot AI Blog）

7. Anthropic 用 Claude Code 两周将百万行 Zig 代码迁移至 Rust

100% 现有测试通过，消耗约 16.5 万美元 API 成本，编译时间从八分钟降至两秒，二进制启动快 6 倍。（来源：AIHOT / Claude Blog）

8. HYPIC：小红书联合北大上交发布混合注意力位置无关缓存

首 token 延迟平均降低 3.25 倍，同 SLO 下可持续 QPS 提升 1.66 倍，任务质量与完全重算仅相差 1.71 分。（来源：AIHOT / 公众号：小红书技术）

💻 开发者/工具（6 个）

1. 面壁智能开源 StaffDeck 企业 AI 数字员工平台

将专业知识、SOP 和决策规则转化为持续工作的数字员工，非传统聊天机器人。（来源：AIHOT / OpenBMB）

2. 开源编程智能体内存方案（SSH 同步）

面向编程 AI 智能体的开源内存项目，支持通过 SSH 同步记忆数据，跨会话保留上下文，可自托管。（来源：AIHOT / HN）

3. MiniMax Code 2.0 桌面端发布

基于 Pi Agent 框架重构，打通恒生金融数据库和企查查 MCP，金融模块即将上线。（来源：AIHOT / 公众号：MiniMax）

4. 百度秒哒 3.5 发布 iOS 打包能力

无代码开发到 iOS 端，无需 Mac/Xcode 即可打包 IPA，内置 SEO Agent，累计超 3500 万用户。（来源：AIHOT / 百度智能云）

5. Decoy 字体：用空间频率混淆对抗 AI 文字抓取

一款 TTF 字体，叠加不同空间频率图形，人眼可读但 AI 读取”诱饵”文字。（来源：AIHOT / HN）

6. LLM cliché highlighter 检测 AI 写作套话

Simon Willison 用 Fable 5 开发，高亮 LLM 文本中常见的十种套话模式。（来源：AIHOT / Simon Willison 博客）
