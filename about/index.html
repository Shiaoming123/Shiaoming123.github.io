关于

你好，我是世傲命。这个博客用来记录前端开发、Node.js、GitHub Pages、写作工作流和个人项目实践中的真实问题。

我把这里当成一份持续更新的技术成长档案：不是只放最后的结论，而是尽量把问题背景、定位过程、取舍原因和可复用的经验都留下来。

技术定位

  • 关注前端工程、交互体验、静态站建设和 AI 工具工作流
  • 偏好把小工具、Demo 和复盘文章组合在一起展示，让读者可以直接体验结果
  • 维护长期可写、可部署、可回滚的个人知识库和博客体系

常用技术栈

  • 前端：HTML、CSS、JavaScript、TypeScript、Vue、React、Astro
  • 工程化：Vite、Node.js、Git、GitHub Pages、Vercel
  • 写作与知识管理：Markdown、Obsidian、静态站发布、图片资产管理
  • AI 辅助：围绕知识库、RAG、token 成本、写作和项目复盘做工具化实践

我希望每篇文章都尽量保留三个东西：

  • 遇到的问题是什么
  • 当时怎么定位和解决
  • 下次可以复用的经验是什么

当前关注

  • 前端基础、页面交互与工程化习惯
  • Node.js、Git、GitHub Pages 等工具链实践
  • Obsidian 写作、图片管理和静态站发布流程
  • 个人 Demo 的设计、部署和文章化复盘

代表项目

  • Tokens Counter：面向 AI 应用开发的 token 与 API 成本估算工具
  • Loading Mind：面向思考过程和研究报告呈现的交互原型
  • 个人博客迁移：从 Hexo 生成产物迁移到 Astro 源码站，并保留 GitHub Pages 静态发布方案

写作原则

  • 写清楚问题发生的环境和限制，不只写“怎么做”
  • 保留关键命令、配置和踩坑点，方便以后复用
  • 能放截图、图表或 Demo 的地方尽量补上可验证的材料
  • 结论保持克制，优先记录自己实际验证过的内容
