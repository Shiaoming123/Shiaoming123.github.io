AIGC 视频来源与版权边界

链接：https://github.com/jnMetaCode/ai-shortfilm-prompts

15 Jun, 2026
来源：wiki/courses/aigc/AIGC视频来源与版权边界.md

这篇适合谁读

这篇适合想系统学习 AIGC 视频创作的人阅读。重点不是堆提示词，而是把来源治理、分镜结构、镜头语言、版权边界和可复用工作流整理成可练习的方法。

本次收集的核心材料

1. jnMetaCode/ai-shortfilm-prompts

链接：https://github.com/jnMetaCode/ai-shortfilm-prompts

本地位置：

内容：

  • Mx-Shell AI 短片提示词方法论。
  • 《丧尸清道夫》相关原始提示词归档。
  • 5 段式结构解释。
  • FAQ、翻车修复案例、通用模板。
  • Claude Code Skill。

最值得读：

  • README.zh.md
  • cheatsheet.zh.md

  • faq.zh.md
  • cases.zh.md
  • templates/multi-shot-narrative.zh.md
  • templates/atmosphere-prefabs.zh.md
  • prompts/README.md
  • prompts/zombie-scavenger.md

版权边界：

  • 方法论、模板、Skill 等整理内容按仓库说明为 MIT。
  • prompts/ 里的 Mx-Shell 原始提示词版权归原作者所有。
  • 适合个人学习、署名引用和方法论参考。
  • 不建议二次公开打包、付费课程搬运、商业训练数据使用。

  • 景别、构图、运镜清单。

  • scripts/validate_prompt.py 自检脚本。

最值得读：

  • README.md
  • SKILL.md
  • references/framework.md

版权边界：

  • 仓库为 MIT。
  • 仓库声明不包含原视频音频、完整字幕、完整转录。
  • 模板为基于公开方法论的重写，不是原视频逐字稿。

原始提示词使用原则

可以做：

  • 私下学习结构。
  • 摘要方法论。
  • 对照原片理解分镜。
  • 署名引用少量片段。
  • 改写成自己的原创题材。

不要做：

  • 把原始提示词全文复制到公开网站。
  • 去掉 Mx-Shell 署名后当作自己的模板卖课。
  • 直接拿含 IP 名的 prompt 做商业项目。
  • 把原始提示词打包成训练数据集。

IP 名称处理

原始材料里出现过一些 IP 或名人风格引用。实际创作时应替换为通用描述。

替换方式：

不建议直接写改写方向
具体影视/游戏 IP 名题材、美术年代、材质、镜头风格
具体角色名原创角色设定
具体名人舞蹈动作节奏、身体部位、舞步特征
真实品牌虚构品牌或无品牌描述

原则：

  • 学结构，不学侵权表达。
  • 学镜头语言，不学具体 IP 外观。
  • 学创作方法，不复制世界观。

知识库保存策略

当前做法：

  • wiki 页面保存结构化学习笔记。
  • external_links 保存公开来源。
  • context 标注本次 Codex 调研日期。
  • 不把原始 prompt 全文复制进知识库页面。

后续可选：

  • 如果要长期离线保存仓库快照，可将 GitHub 仓库作为原始资料放入 raw/github project/aigc/。
  • 如果要做自己的作品，应另建 wiki/courses/aigc/项目名.md 或 wiki/ideas/项目名.md 记录创作过程。

后续更新规则

当新增 AIGC 视频学习来源时，记录：

  • 来源名称。
  • URL。
  • 作者。
  • 发布时间或检查日期。
  • 材料类型：原片、教程、prompt、工具、模型、后期流程。
  • 可复用程度：一次性参考 / 长期方法论 / 工具入口。
  • 版权边界：可复制、可引用、仅学习、需授权。

不要把短视频平台的二手搬运当作最终证据。优先回到：

  • 原作者主页。
  • 官方仓库。
  • 原始教程视频。
  • 项目文档。
  • 模型或平台官方说明。
